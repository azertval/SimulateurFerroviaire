GeoParser — Pipeline principal

Pipeline complet de transformation GeoJSON → modèle ferroviaire.

Tâche asynchrone

GeoParsingTask lance GeoParser dans un thread détaché. La communication vers l'UI passe exclusivement par PostMessage :

Message Contenu
WM_PROGRESS_UPDATE Avancement 0–100
WM_PARSING_SUCCESS Parsing terminé
WM_PARSING_ERROR Pointeur std::string* (à libérer par le destinataire)

Pipeline global

Voir GeoParser::parse() pour l'implémentation complète.

Phase Classe Description
1–2 GraphBuilder Chargement GeoJSON, projection WGS-84 → UTM, snap + fusion des nœuds
3–5 TopologyExtractor Détection des aiguillages (degré == 3), extraction des StraightBlock, câblage
6 SwitchOrientator Orientation root / normal / deviation, calcul des tips CDC
7–8 DoubleSwitchDetector Détection des doubles aiguilles, absorption du segment de liaison, validation CDC
9 GeoParser Clear + transfert vers TopologyRepository
10 GeoParser Résolution des pointeurs partenaires, branches, voisins + buildIndex()

‍Phase 3 — Détection des aiguillages : seuls les nœuds de degré exactement 3 (jonction en Y) sont des SwitchBlock. Les nœuds de degré > 3 (croisements en X) sont des frontières de découpe pour les StraightBlock mais ne génèrent pas d'aiguillage.

‍Phase 10 — Résolution des pointeurs : après transfert en unique_ptr, les adresses sont stables. Chaque bloc reçoit ses pointeurs directs via setBranchPointers() / setNeighbourPointers(), puis buildIndex() construit les tables de lookup O(1).

Classes du pipeline

Classe Rôle
GeoParser Orchestrateur du pipeline
GeoParsingTask Exécution asynchrone (évite le blocage UI)
GeometryUtils Projection UTM, interpolation, angles, snap de grille
GraphBuilder Construction du graphe depuis le GeoJSON (phases 1–2)
TopologyGraph Graphe planaire non-orienté (nœuds + arêtes), union-find
TopologyEdge Arête avec géométrie polyligne métrique et longueur planaire
TopologyExtractor Extraction des blocs depuis le graphe (phases 3–5)
SwitchOrientator Orientation géométrique (phases 6, 6b, 6c, 6d)
DoubleSwitchDetector Détection des doubles aiguilles et validation CDC (phases 7–8)

Éléments interactifs

Classe Rôle
InteractiveElement Interface abstraite commune (id, type). Logger statique partagé. Copie interdite, déplacement autorisé
ShuntingElement Étend avec un état opérationnel + helpers isFree() / isOccupied() / isInactive()
StraightBlock Tronçon de voie droite. Longueur géodésique Haversine. Voisins résolus via StraightNeighbours
SwitchBlock Aiguillage 3 branches. Orientation + tips CDC + support double aiguille + état ActiveBranch

Hiérarchie

InteractiveElement getId(), getType(), m_logger (static)
└── ShuntingElement getState() [FREE / OCCUPIED / INACTIVE]
├── StraightBlock
└── SwitchBlock

‍Logger statique : InteractiveElement::m_logger est partagé par toutes les instances → un seul fichier Logs/InteractiveElements.log.

Champ Type Description

**SwitchBlock::SwitchBranches**

Champ Type Description
root ShuntingElement* Bloc sur la branche root
normal ShuntingElement* Bloc sur la branche normale
deviation ShuntingElement* Bloc sur la branche déviée

État opérationnel des aiguillages

Méthode Description
SwitchBlock::getActiveBranch() Retourne ActiveBranch::NORMAL ou DEVIATION
SwitchBlock::isDeviationActive() Raccourci booléen
SwitchBlock::setActiveBranch(branch) Assigne l'état + propage aux partenaires
SwitchBlock::toggleActiveBranch() Alterne + propage + retourne le nouvel état
