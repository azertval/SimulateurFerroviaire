◆ create()

void PCCPanel::create ( HWND  hParent,
HINSTANCE  hInstance 
)

Enregistre la classe Win32 et crée la fenêtre enfant masquée.

La fenêtre est positionnée à (0, 0) avec les dimensions de la zone cliente de hParent. Elle est masquée (SW_HIDE) jusqu'au premier appel à toggle.

L'enregistrement de la classe est idempotent : si RegisterClassExW retourne ERROR_CLASS_ALREADY_EXISTS, l'erreur est ignorée.

Parameters
hParentHandle de la fenêtre parente (MainWindow).
hInstanceHandle de l'instance Win32 de l'application.
Exceptions
std::runtime_errorSi CreateWindowExW retourne nullptr.

Gestionnaire de WM_PAINT — délègue le dessin à TCORenderer.

Ouvre un BeginPaint / EndPaint, interroge GetClientRect pour obtenir les dimensions courantes, puis appelle TCORenderer::draw. Aucune logique de dessin n'est présente ici.

◆ refresh()

void PCCPanel::refresh ( )

Force un rafraîchissement du dessin TCO.

Invalide l'intégralité du rectangle client via InvalidateRect, ce qui provoque un WM_PAINT au prochain cycle de messages. N'effectue rien si le panneau est masqué ou non créé.

À appeler depuis MainWindow::onParsingSuccess pour mettre à jour le schéma après chargement d'un fichier GeoJSON.

◆ resize()

void PCCPanel::resize ( )

Redimensionne le panneau pour couvrir toute la zone cliente du parent.

Interroge GetClientRect sur la fenêtre parente et applique les dimensions via SetWindowPos. À appeler depuis le gestionnaire WM_SIZE de MainWindow.

Aucune action si create n'a pas été appelé.

◆ toggle()

void PCCPanel::toggle ( )

Alterne la visibilité du panneau (masqué ↔ visible).

Lors d'une transition masqué → visible, redimensionne d'abord le panneau via resize, puis invalide le rectangle pour déclencher un WM_PAINT immédiat. Aucune action si create n'a pas été appelé.
